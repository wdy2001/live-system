# 生活缴费系统 - Product Requirement Document

## Overview
- **Summary**: 本项目是一个面向居民用户的生活缴费服务系统，提供电费、水费、燃气费的在线缴费、账单查询、计费规则查询以及故障报修等核心功能。系统采用前后端分离架构，后端基于 Python + MySQL，前端使用 Vue.js，支持完整的用户交互体验。
- **Purpose**: 解决居民线下缴费排队繁琐、账单管理不便、故障报修流程复杂等问题，提供一站式线上生活服务入口，提升居民生活便利性和服务效率。
- **Target Users**: 城市居民用户、物业管理人员、社区服务人员

## Goals
- 提供电费、水费、燃气费三种生活缴费的在线支付功能
- 支持分类查看历史缴费记录，支持多维度筛选和查询
- 提供透明的计费规则展示，让用户清楚费用计算方式
- 提供故障报修入口，支持用户提交报修申请并跟踪处理进度
- 系统响应快速，界面简洁易用，操作流程顺畅

## Non-Goals (Out of Scope)
- 不涉及真实第三方支付渠道对接（使用模拟支付流程）
- 不包含用户实名认证和身份审核流程
- 不涉及实时抄表数据接入（使用模拟账单数据）
- 不包含管理员后台功能（仅用户端功能）
- 不包含移动端 App 开发（仅 Web 端响应式布局）
- 不包含消息推送、短信通知等第三方服务集成

## Background & Context
- 随着智慧城市建设的推进，居民对线上公共服务的需求日益增长
- 传统线下缴费模式效率低、体验差，已无法满足现代生活需求
- 本项目作为学习和演示项目，展示完整的 Web 应用开发流程
- 技术选型：后端 Python FastAPI（高性能、自动文档、类型安全），前端 Vue 3 + Element Plus（成熟生态、开发效率高），数据库 MySQL（关系型、稳定可靠）

## Functional Requirements

### FR-1: 用户认证
- 用户可以注册账号（用户名、密码、手机号）
- 用户可以登录/登出系统
- 登录状态保持和会话管理
- 密码加密存储

### FR-2: 电费缴费
- 用户可以查看当前电费账单（户号、用电量、应缴金额、欠费状态）
- 用户可以输入缴费金额进行电费缴费
- 支持多户号管理（用户可添加多个电表户号）
- 缴费成功后更新账单状态并生成缴费记录

### FR-3: 水费缴费
- 用户可以查看当前水费账单（户号、用水量、应缴金额、欠费状态）
- 用户可以输入缴费金额进行水费缴费
- 支持多户号管理（用户可添加多个水表户号）
- 缴费成功后更新账单状态并生成缴费记录

### FR-4: 燃气缴费
- 用户可以查看当前燃气账单（户号、用气量、应缴金额、欠费状态）
- 用户可以输入缴费金额进行燃气费缴费
- 支持多户号管理（用户可添加多个燃气户号）
- 缴费成功后更新账单状态并生成缴费记录

### FR-5: 缴费记录查询
- 用户可以查看所有历史缴费记录列表
- 支持按缴费类型筛选（电费/水费/燃气费/全部）
- 支持按时间范围筛选
- 支持按缴费状态筛选（成功/失败/处理中）
- 单条记录详情查看（金额、时间、户号、交易单号等）

### FR-6: 计费规则展示
- 展示电费计费规则（阶梯电价、峰谷电价说明）
- 展示水费计费规则（阶梯水价、污水处理费说明）
- 展示燃气费计费规则（阶梯气价说明）
- 规则按类型分类展示，清晰易懂

### FR-7: 故障报修
- 用户可以提交故障报修申请（选择故障类型、描述问题、上传位置/地址、联系方式）
- 故障类型：电力故障、供水故障、燃气故障
- 用户可以查看自己的报修记录列表
- 用户可以查看报修单处理状态（待处理/处理中/已完成）
- 报修单详情查看（包含处理进度备注）

## Non-Functional Requirements

- **NFR-1**: 页面加载响应时间 < 2 秒，API 响应时间 < 500ms（95% 请求）
- **NFR-2**: 前端界面支持响应式布局，适配桌面端和移动端浏览器
- **NFR-3**: 密码使用 bcrypt 加密存储，敏感操作需验证用户身份
- **NFR-4**: 代码结构清晰，模块化设计，便于后续功能扩展
- **NFR-5**: 数据库表设计合理，建立适当索引保证查询性能
- **NFR-6**: 项目结构完整，包含 README 文档和启动说明

## Constraints

- **Technical**:
  - 后端语言：Python 3.9+
  - 后端框架：FastAPI
  - 前端框架：Vue 3 + Element Plus
  - 数据库：MySQL 8.0+
  - ORM：SQLAlchemy
  - 代码托管：GitHub
- **Business**:
  - 开发周期：10 个工作日
  - 项目性质：学习/演示项目
- **Dependencies**:
  - FastAPI, SQLAlchemy, pymysql, bcrypt, python-jose
  - Vue 3, Vue Router, Pinia, Element Plus, Axios
  - MySQL 数据库服务

## Assumptions

- 用户通过用户名和密码登录，无需短信验证
- 支付流程为模拟支付，不接入真实支付网关
- 账单数据为系统预置的模拟数据，用户缴费后实时更新
- 单用户可管理多个户号（电表、水表、燃气表各多个）
- 故障报修由系统模拟处理进度更新，无需真实工单系统对接
- 部署时用户自行配置 MySQL 数据库连接信息

## Acceptance Criteria

### AC-1: 用户注册与登录
- **Given**: 用户访问系统注册页面
- **When**: 用户填写用户名、密码、手机号并提交注册
- **Then**: 系统创建用户账号，密码加密存储，提示注册成功并跳转到登录页
- **Verification**: `programmatic`

### AC-2: 用户登录认证
- **Given**: 已注册用户在登录页输入正确的用户名和密码
- **When**: 用户点击登录按钮
- **Then**: 系统验证通过，生成访问令牌，跳转到系统首页
- **Verification**: `programmatic`

### AC-3: 电费账单查看与缴费

- **Then**: 显示当前绑定户号的电费账单信息（用电量、应缴金额、状态）
- **Verification**: `programmatic`

### AC-4: 电费缴费流程
